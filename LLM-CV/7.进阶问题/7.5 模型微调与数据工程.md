# 模块五：模型微调与数据工程 (Fine-Tuning & Alignment)

> **面试官视角与定位：**
> 区分“只会调用第三方 API 的上层应用工程师”与“具备全栈大模型定制能力、能打造特定领域硬核壁垒的高级/资深工程师”。核心考察候选人对 **PEFT 轻量化微调机制**、**高质量数据清洗流** 以及 **DPO / Preference Alignment 对齐技术** 的掌握程度。

### 核心考察问题与深挖路线

#### 5.1 轻量化微调技术原理与显存优化 (PEFT & LoRA)

* **面试官提问：**
> “在显存资源有限的情况下，我们通常会采用 LoRA 或 QLoRA 进行微调。请讲讲 LoRA 的数学原理是什么？为什么它能大幅降低训练时的显存开销？参数 Rank $r$ 和 Alpha 对微调效果有什么影响？”


* **追问路线：**
* *追问 1（底层细节）：* QLoRA 相比于传统 LoRA 做了哪些突破性改进？（如 NF4 量化数据类型、双量化 Double Quantization、分页优化器 Paged Optimizers）
* *追问 2（部署合并）：* 微调完成后，我们在线上部署时，需要把 LoRA 权重与 Base Model 合并（Merge）吗？为什么？如果不合并会有什么后果？


* **参考答案/关键考点：**
* **LoRA 数学原理：** 冻结原模型权重 $W_0 \in \mathbb{R}^{d \times k}$，通过旁路引入两个低秩矩阵 $A \in \mathbb{R}^{r \times k}$ 与 $B \in \mathbb{R}^{d \times r}$（其中 $r \ll \min(d, k)$）来近似权重更新：$W = W_0 + \frac{\alpha}{r} BA$。显存节省来自于**无需保存和更新全量参数的梯度（Gradients）与优化器状态（Optimizer States）**。
* **QLoRA 突破：** 引入 **NF4 (NormalFloat 4)** 数据类型（针对正态分布权重优化的 4-bit 量化）、**Double Quantization**（对量化常数再次量化以省显存）、**Paged Optimizers**（借助 CUDA 统一内存解决显存峰值 OOM）。
* **权重合并：** 生产环境中通常将旁路权重合并回基座模型（$\Delta W = \frac{\alpha}{r} BA$ 加回 $W_0$），从而避免推理时增加额外的旁路计算延迟；若需要单模型多任务切换，则保持 LoRA 旁路独立并使用 Dynamic LoRA Serving 框架（如 vLLM 的 Multi-LoRA）。

#### 5.2 SFT 数据工程与清洗 (Data Engineering for SFT)

* **面试官提问：**
> “业界常说‘数据质量决定了微调的上限’。如果让你为公司特定业务（如法律合同审查或客服导购）准备 10,000 条 SFT（指令微调）数据，你如何设计从原始数据收集、清洗、扩充到去重的全流程？”


* **追问路线：**
* *追问：* 针对模型微调后容易出现的“灾难性遗忘（Catastrophic Forgetting）”或“过拟合（Overfitting）”现象，你在数据构筑和训练参数配置上有什么预防手段？


* **参考答案/关键考点：**
* **数据流水线：**
1. **数据清洗与抽取：** 正则提纯、格式规范化（转换为 Alpaca / ShareGPT 统一格式）、敏感词与 PII 脱敏。
2. **去重（Deduplication）：** 基于 MinHash / LSH（局部敏感哈希）或向量相似度删除高度重复的语料。
3. **数据扩增（Data Synthesis）：** 利用强模型（如 GPT-4o / Qwen-Max）通过 Self-Instruct / Evol-Instruct 范式生成多样化的 Instruction-Response 对。


* **防遗忘策略：** 在 SFT 数据集里混合 10%~20% 的通用高质量指令数据（如 ShareGPT 数据集）；设置较小的 Learning Rate（如 $10^{-5}$ 到 $10^{-4}$），配合 Cosine 学习率衰减，训练 Epoch 通常只跑 2~3 轮。

#### 5.3 偏好对齐与 DPO (Preference Alignment & DPO)

* **面试官提问：**
> “如果 SFT 后的模型回答依然不够安全或不够符合业务偏好，我们需要做偏好对齐。为什么现在业界更倾向于使用 DPO (Direct Preference Optimization) 而不是传统的 RLHF (PPO)？”


* **追问路线：**
* *追问：* DPO 的损失函数（Loss Function）核心逻辑是什么？它在构造数据对（Chosen vs Rejected）时有什么要求？


* **参考答案/关键考点：**
* **DPO vs PPO：** PPO 需要同时训练 Actor、Critic、Reward Model 和 Reference Model 四个模型，工程实现极其复杂、显存开销巨大且极难收敛；DPO 通过数学推导，证明了**隐式奖励函数可以由 policy 模型和 reference 模型之间的 Logits 概率比直接表示**，将强化学习转化为了一个简单的分类交叉熵 Loss，无需单独训练 Reward Model。
* **数据要求：** 需要包含 $(Prompt, Y_{chosen}, Y_{rejected})$ 三元组数据，即针对同一个 Prompt，明确指定哪个回答更好（Chosen），哪个回答更差（Rejected）。
