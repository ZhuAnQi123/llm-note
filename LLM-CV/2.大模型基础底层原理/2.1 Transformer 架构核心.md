### Q1：请手写/推导自注意力机制（Self-Attention）的计算公式，并解释为什么公式中要除以 $\sqrt{d_k}$？如果不除会发生什么？

**🎯 核心考点：Transformer 的数学基石，Softmax 梯度消失原理。**

**参考答案：**

1. **计算公式：**
自注意力机制的矩阵形式公式为：

$$Attention(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$
$$\text{注意力输出} = \text{转化为百分比的（(查询相似度得分) / 缩放因子）} \times \text{具体内容}$$



其中 $Q$（Query问题）、$K$（Key库中的标签）、$V$（Value,具体内容）是由输入向量通过不同的权重矩阵 $W_Q, W_K, W_V$ 线性变换得到的，$d_k$ 是 Key 向量的维度（即缩放因子）。
2. **为什么要除以 $\sqrt{d_k}$（缩放因子的作用）：**
* **防止梯度消失：** 当 $d_k$（维度）非常大时，$QK^T$ 点积的点积值通常会变得很大。这会导致经过 $\text{softmax}$ 函数计算后，概率分布会变得一家独大（即绝大部分权重集中在某一个 Token 上，其余接近于 0）。
* **数学原理：** 假设 $Q$ 和 $K$ 的每个分量都是均值为 0、方差为 1 的独立随机变量，那么它们点积的均值为 0，方差则为 $d_k$。为了将方差重新缩放回 1，使其保持平稳的分布，必须除以标准差 $\sqrt{d_k}$。
* **后果：** 如果不除以 $\sqrt{d_k}$，$\text{softmax}$ 就会进入饱和区，此时**梯度（Gradient）会趋近于 0**，导致模型在反向传播时无法有效地进行参数训练（梯度消失）。

### Q2：请对比 Encoder-only（如 BERT）、Decoder-only（如 GPT、DeepSeek）和 Encoder-Decoder（如 T5）三种架构的区别，并说明为什么目前大语言模型（LLM）几乎全网倒向了 Decoder-only 架构？

**🎯 核心考点：模型架构演进洞察力。**

**参考答案：**

1. **三种架构的区别：**

| 架构类型 | 核心超能力 | 视线范围 | 典型代表 | 相当于... | 适合做 |
| --- | --- | --- | --- | --- | --- |
| **Encoder-only** | **做理解** (Understand) | 左右全看（双向） | BERT | 逐字分析的**阅读理解考生** | 文本分类/情感分析/重点提取 |
| **Decoder-only** | **做生成** (Generate) | 只能看左边（单向） | GPT系列, DeepSeek系列, Llama | 边想边说的**小说家** | 自回归生成任务 |
| **Encoder-Decoder** | **做转换** (Translate) | 左边全看 $\rightarrow$ 右边单向 | T5, BART | 听完一段话后立刻用另一种语言复述的**翻译官** | 翻译/纠错/摘要 |

2. **为什么目前 LLM 全面转向 Decoder-only（核心原因）：**
* **自回归与涌现能力（Emergent Abilities）：** 随着参数量跨越百亿（10B）门槛，Decoder-only 架构在 Zero-shot、Few-shot 和复杂推理上展现出了惊人的“涌现”能力。而 Encoder-Decoder 在超大参数量下的泛化能力反而不如纯 Decoder。
* **零样本泛化更强（提示词工程的基石）：** Decoder-only 将“上下文”和“生成目标”放在同一个序列里统一处理，天生完美契合 In-context Learning（语境学习）。你只需要喂一句话，它就能往下续写，不需要像传统架构那样对特定任务做精细的接口微调。
* **计算效率与显存优化：** Decoder-only 结构更简单，在线上推理时，可以非常完美地利用 **KV Cache（键值缓存）** 避开历史 Token 的重复计算，极大地优化了高并发下的工程吞吐量。

### Q3：什么是 RoPE（旋转位置编码，Rotary Position Embedding）？它相比于传统的绝对位置编码（如 BERT 的可学习编码）有什么绝对优势？

**🎯 核心考点：现代大模型（LLM）实现长文本（Long Context）的底层核心技术。**

**参考答案：**

1. **RoPE 的核心思想：**
RoPE 是一种将位置信息注入大模型的方法。它通过**外积复数旋转矩阵**的方式，将位置编码应用在 $Q$ 和 $K$ 向量上。其巧妙之处在于：虽然它是在三维/高维空间中对 $Q$ 和 $K$ 进行“旋转”（形式上像绝对位置编码），但在它们进行点积操作（$QK^T$）时，旋转结果会自动抵消，**最终保留下来的纯粹是两个 Token 之间的相对距离（Relative Distance）信息**。
2. **相比传统编码的绝对优势：**
* **完美的相对位置感知：** 传统的绝对位置编码（如 GPT-2、BERT）只告诉模型“我是第 5 个 Token”，模型很难直接算出来第 5 个和第 100 个的相对距离。而 RoPE 天然具备因果相对位置衰减特性（距离越远，注意力权重自然下降）。
* **长文本外推性（Extrapolation）：** 传统的绝对位置编码在训练时如果只看过了 2K 长度，遇到 8K 的输入时模型就会彻底崩溃（因为没见过第 2001 个位置的编码）。而 RoPE 由于是基于角度旋转的连续数学函数，可以通过 **Linear Scaling（线性内插）** 或 **YARN** 等技术，在不需要重新训练的情况下，直接把模型的上下文窗口从 4K “外推”扩张到 32K 甚至 128K。

### Q4：在 Transformer 中，前馈神经网络（FFN）层在现代 LLM 中发生了什么演进？什么是 SwiGLU，为什么现在的大模型（如 Llama、Qwen、DeepSeek）都用它替代了传统的 ReLU/GELU？

**🎯 核心考点：模型内部微观结构的升级，大厂参数调优细节。**

**参考答案：**

1. **演进路径：** 传统的 Transformer 里的 FFN 是简单的两层全连接：`Linear -> ReLU/GELU -> Linear`。而现代 LLM 几乎全盘采用了 **GLU（门控线性单元，Gated Linear Unit）** 的变体，最典型的是 **SwiGLU**。
2. **什么是 SwiGLU：**
它引入了一个“门控（Gate）”机制。它将输入同时喂给两个不同的线性层，其中一路经过 **Swish** 激活函数（也称 SiLU），然后与另一路的结果进行**元素级相乘（Hadamard Product）**，最后再通过一个线性层输出。
3. **为什么都用它（优缺点）：**
* **优势：** SwiGLU 的数学梯度更加平滑，且“门控”机制允许模型在传递信息时，自主控制让多少比例的信息流向下游。实验证明，换装 SwiGLU 后，大模型在各项 Benchmark 上的收敛速度和最终表现都**显著优于**传统的 ReLU 或 GELU。
* **工程代价：** SwiGLU 增加了约 $1.5$ 倍的参数量和计算量（因为它多用了一个线性投影矩阵），但大厂普遍认为“用这点算力换取模型能力的巨大提升”是极其划算的。

### Q5：什么是 Multi-Query Attention (MQA) 和 Grouped-Query Attention (GQA)？它们是为了解决 Multi-Head Attention (MHA) 的什么工程痛点而设计出来的？

**🎯 核心考点：大模型高并发推理（LLM Inference）优化的终极必考题。**

**参考答案：**
这些变体全是为了解决 **MHA 架构下 KV Cache 占用显存过大、导致推理时 Batch Size 无法扩大的致命痛点（Memory-bound）**。

[Image comparison of MHA, MQA, and GQA structures]

1. **Multi-Head Attention (MHA，传统多头注意力)：**
* *每个* Query Head 都有自己独立的 Key Head 和 Value Head。如果模型有 32 个头，那每次生成都要存 32 份 KV 显存。


2. **Multi-Query Attention (MQA，多查询注意力)：**
* **极端优化：** 让*所有*的 Query Head 共同共享**同一个** Key Head 和 Value Head。
* *优势：* KV Cache 的显存占用直接暴跌到原来的 $1/32$，大大释放了显存，允许线上服务开极大的 Batch Size。
* *劣势：* 强行让模型的所有头看同一份 KV，会导致模型表达能力严重受损，任务表现力有所下降。


3. **Grouped-Query Attention (GQA，分组查询注意力——现代 LLM 的行业标准)：**
* **折中方案：** 将 Query Head 进行分组（比如 32 个头分成 8 个组，每组 4 个头）。**同一个组内的 Query 共享同一个 Key 和 Value Head**。
* *总结：* GQA 完美兼顾了 MHA 的“高模型质量”和 MQA 的“极高推理速度与省显存”，是目前 Llama 3、Mistral 等开源霸主模型的标配。

## 名词掌握
* **Self-Attention**   让序列中每个词都去“关注”其他所有词，并计算它们之间相关性强弱的机制。
* **Softmax** 把一组分数（相似度）转换成概率分布（总和为1）的函数，让注意力权重可解释。
* **自回归生成** 每次生成一个词，并把新生成的词拼回输入，再继续生成下一个词，循环往复。
* **KV Cache（键值缓存）** 把已经算好的历史 Token 的 K 和 V 存下来，生成新词时直接复用，避免重复计算。
* **模型参数（权重）**	训练出来的海量数字矩阵，是模型的“长期记忆”（固化知识），存在 GPU 显存里。
* **GPU 显存（VRAM）** 显卡自带的超高速内存，用于存放模型参数、KV Cache 和中间计算结果。
* **RoPE（旋转位置编码）**	通过旋转矩阵把位置信息注入 Q 和 K，让点积结果天然带上相对距离信息。
* **FFN** Transformer 里的“深度思考”模块，负责对注意力层输出的信息做非线性变换和推理。
这 5 道题的硬核程度非常高，尤其是 **RoPE 的外推** 和 **GQA 的省显存原理**，是面试大厂算法工程岗或高级 AI 开发时的必杀题。
* **GLU（门控线性单元）** 引入一个“门”来控制信息流通量的 FFN 结构变体。
* **SwiGLU** 使用 Swish（SiLU）作为门控激活函数的 GLU 变体，梯度更平滑，效果优于 ReLU/GELU。
* **MHA（多头注意力）**	每个注意力头都有自己的 Q、K、V 投影，32 个头就有 32 份独立的 KV 缓存。
* **MQA（多查询注意力）**	所有 Query Head 共享同一份 Key 和 Value，KV 显存暴降到 1/32，但模型表达能力下降。
* **GQA（分组查询注意力）**	将 Query Head 分组，同组内的头共享一份 KV，既省显存又保住模型质量（Llama 3 标配）。
* **KV Head**	负责生成 K 和 V 向量的那部分网络结构，每个注意力头可以有自己的，也可以共享。

* **建议：** 重点消化一下 **Q1（除以 $\sqrt{d_k}$ 控梯度）** 和 **Q5（GQA 优化推理显存）** 的逻辑，能用大白话讲给面试官听。
