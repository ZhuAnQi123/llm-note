# 2.2 LLM 核心参数调节

在调用任何大模型 API（如 OpenAI、DeepSeek）或者部署开源模型（如 Llama、Qwen）时，你都会面对一堆控制输出文本的参数。面试官喜欢通过这些参数的底层数学原理，来考察你**是否真正理解模型的生成机制，以及是否有能力针对“严谨的客服场景”和“发散的文案创意场景”做精细化调优**。

---



### Q1：大模型在生成文本时，Temperature（温度）参数的数学原理是什么？为什么把 Temperature 调高，模型就会变得更具创造力（胡说八道）？

**🎯 核心考点：Softmax 函数变形，概率分布平滑度控制。**

**参考答案：**

1. **数学原理：**
在大模型解码（Decoding）的最后阶段，模型会输出一个包含所有 Token 原始分数的向量（称为 Logits $z_i$）。通常我们会使用 $\text{softmax}$ 函数将其转化为概率分布。当引入 Temperature（记为 $T$）后，计算公式变形为：

$$P(x_i) = \frac{\exp(z_i / T)}{\sum_{j} \exp(z_j / T)}$$

说白了这个公式就是把模型给每个词打的原始分，通过 $T$ 放大或缩小差距，再用 $\exp$ 变成正数并拉开档次，最后除以所有词的总得分，算出每个词最终的出场概率。

* 分子—— $\exp(z_i / T)$
    * $z_i$ 是模型给第 $i$ 个词打的**原始分数（Logits）**。比如“米饭” 10 分，“面条” 8 分，“石头” -5 分。  
    * $\exp(x)$（即 $e^x$） 是指数函数。一是可以把所有的分数转换为正数；二是可以指数级地放大差距
* 分母—— $\sum_{j} \exp(z_j / T)$
    * $\sum$（求和符号）是为了把所有值都加起来因为我们到时候要算得是概率，计算目标占所有值之和的百分比
    * $T$ 是分母的分母（在 $z_i / T$ 里），它就像是一个“贫富差距调节器”。   
        $T$很大的时候除一个很大的分母差距再用指数计算就变小了。-- 这时候有更有创造力。 
        $T$小的时候相反，值就比较固定



2. **为什么调高 $T$ 会增加“创造力”/“随机性”：**
* **当 $T \to 0$ 时（趋近于 0）：** 分母中的 $z_i / T$ 会把最高分和次高分的差距无限放大。最终 $\text{softmax}$ 会退化为一个 **One-hot 向量**（概率最高的 Token 变成 1，其余全变成 0），这被称为 **贪婪搜索 (Greedy Search)**。模型每次都会选一模一样的词，结果完全确定。
* **当 $T > 1$（调高温度）时：** $z_i / T$ 会使原本高分和低分的 Logits 之间的差异**被无形中缩小、被压平**。通过公式计算后的概率分布会变得非常**平滑（Uniform）**。原本那些概率只有 1% 的冷门词、边缘词，其概率可能会被抬升到 5% 甚至 10%。
* **结果：** 采样时模型选中这些非首选词的概率大大增加，因而输出显得更有“创意”和不可预测性。但代价是：温度过高（如 $T \ge 1.5$）会导致概率分布彻底混乱，模型开始疯狂吐出乱码或完全不合逻辑的话。

### Q2：请解释 Top-k 采样和 Top-p (Nucleus Sampling) 采样的区别。线上高并发业务中更推荐哪一种？为什么？

**🎯 核心考点：动态截断 vs 静态截断，截断采样的边界控制。**

**参考答案：**
为了防止模型由于平滑概率分布而采样到极度离谱的词，我们需要对概率分布进行“截断（Truncation）”。

1. **Top-k 采样（数量截断）：**
* *机制：* 无论概率怎么分布，固定在概率最高的**前 $k$ 个** Token中采样（采样就是随机抽取）。
* *缺点：* 它是静态的。如果前 2 个词的概率已经占了 99%（模型很笃定），你非要保留前 50 个词（$k=50$），后面 48 个就是废话，容易跑题。


2. **Top-p 采样（累积概率截断 / 核采样）：**
* *机制：* 将所有 Token 按概率从大到小排序，然后从前往后累加概率，当累加的概率之和刚刚达到或超过门槛 $p$（如 $p=90%$）时停止，用这个动态集合（核）内的 Token 采样。比如模型很笃定时（“一加一等于_”，候选词“二”占 99%），保留前90%名单里就只有“二”这一个词；模型纠结时，名单会自动拉长。
* *优势：* 它是**动态自适应**的。当模型有把握时，这个集合可能只包含 1~2 个词；当模型没把握、很困惑时，集合会自动扩大，包含几十个词。


3. **线上业务推荐：**
更推荐使用 **Top-p 采样**（配合 $T$ 调节）。各大厂商的 API 实践表明，Top-p 能在保持生成文本丰富度的同时，提供比 Top-k 更加稳定的句子连贯性。
* *避坑警示：* 工业界规范中，**通常只调节 Temperature 和 Top-p 中的一个**（如果调 Temperature，就把 Top-p 设为 1.0），不要两个同时高频改动，否则会导致概率空间极难追踪和 debug。

### Q3：什么是 Repetition Penalty（重复惩罚）和 Presence Penalty（存在惩罚）？在构建长文本生成或对话客服时，应该如何调节它们？

**🎯 核心考点：缓解大模型“复读机”现象的工程调参手段。**

**参考答案：**
大模型在生成长文本或陷入某种思维定势时，很容易进入局部死循环，变成“复读机”（不断重复生成同一段话）。

1. **Repetition Penalty（重复惩罚）：**
* *数学机制：* 直接干预 Logits。在预测下一个 Token 时，检查之前已经生成过的 Token。如果某个 Token 已经出现过，就**直接将其原始得分 $z_i$ 除以一个惩罚系数 $\alpha$**（通常 $\alpha > 1.0$，如 1.1 或 1.2）。
* *作用：* 强行降低已出现词的再次被选中的概率。


2. **Presence Penalty 与 Frequency Penalty（常用于 OpenAI API）：**
* **Frequency Penalty（频率惩罚）：** 惩罚力度与该词**出现的次数成正比**。一个词出现得越多，下一次被惩罚得越惨，鼓励模型多换词。适合研究报告、会议纪要总结等
* **Presence Penalty（存在惩罚）：** **一视同仁。** 只要这个词出现过（不管是一次还是十次），就施加一个固定的惩罚值。它更倾向于鼓励模型**开拓全新的话题或引入新概念**。适合小红书爆款营销文案、创意小说写作、头脑风暴、角色扮演对话


3. **业务调节建议：**
* **严禁过度惩罚：** 惩罚系数千万不能设得太高（例如 Repetition Penalty 设到 1.5 以上）。否则会导致模型为了刻意避开已经出现过的常用字（如“的”、“是”、“我们”），被迫去使用一些生僻词、火星文甚至直接乱码。通常设在 `1.05 ~ 1.12` 之间即可。

### Q4：【实战选型题】公司现在有两个落地业务：业务 A 是“自动化法律合同审查与条款纠错”；业务 B 是“小红书爆款营销文案创作”。请为这两个业务设计最合理的参数组合。

**🎯 核心考点：参数调节与真实业务场景的对齐能力。**

**参考答案：**

#### 业务 A：自动化法律合同审查（要求：绝对严谨、确定性、零胡编乱造）

* **Temperature 设定：** `0.0` 或极其接近 `0`（如 `0.1`）。
* **Top-p 设定：** `1.0`（不做动态截断，因为温度设为 0 已经是贪婪搜索，Top-p 失去意义）。
* **Repetition Penalty：** `1.0`（保持默认，不进行干预。因为法律文本中许多专有名词如“甲方”、“乙方”、“违约责任”必须高频重复出现，惩罚会导致术语变形）。
* **选型理由：** 确保输入相同的合同文本，模型每一次给出的修改意见、法律依据和条文引用都是完全确定、标准且一致的，把幻觉率降到物理最低。

#### 业务 B：小红书爆款营销文案创作（要求：网感、创意、风格多变、防同质化）

* **Temperature 设定：** `0.7 ~ 0.9` 之间。
* **Top-p 设定：** `0.85 ~ 0.9` 左右（配合温度，保留一定比例的低概率网感词彙，但切掉极度离谱的乱码词）。
* **Presence Penalty：** 可轻微调高至 `0.2 ~ 0.4`。
* **选型理由：** 较高的温度和存在惩罚能够打破模型的线性思维，促使模型输出“吸睛”、“打破常规”的修辞和文风。同时即使用户反复点击“重新生成”，也能得到完全不同视角的几版营销文案，完美满足创意需求。
