# 5.2 RAG 系统专项评估

## Q4：请详细阐述 Ragas 评估框架中的“RAG 三元组”四大核心指标（忠实度、答案相关性、上下文召回率、上下文精确度）的数学/逻辑含义。

**🎯 核心考点：RAG 管道各阶段问题的精准定位。**

**参考答案：**
Ragas 框架通过评估 **Query (Q)、Context (C)、Response (R)** 三者之间的交叉关系，设计了以下四大核心指标：

1. **忠实度 (Faithfulness - 测 R 选自 C)：**
* *含义：* 评估大模型最终生成的回答（R）中，所有的事实陈述是否都能在召回的上下文（C）中找到依据。
* *计算逻辑：* 让大模型将 R 拆解为若干个独立的事实陈述句，逐一判断这些句子是否被 C 包含。$\text{得分} = \frac{\text{被支持的事实陈述数}}{\text{总事实陈述数}}$。低于 0.8 说明大模型在**基于材料瞎编（产生内在幻觉）**。


2. **答案相关性 (Answer Relevance - 测 R 针对 Q)：**
* *含义：* 评估最终回答（R）是否直接、切题地响应了用户的提问（Q），有没有答非所问或扯闲话。
* *计算逻辑：* 让大模型根据生成的 R 反向推导可能的问题 $Q_{pred}$，计算 $Q_{pred}$ 与真实 $Q$ 的向量相似度。


3. **上下文精确度 (Context Precision - 测 C 针对 Q 的排序)：**
* *含义：* 评估系统召回的那些文档块（C）中，真正有用的黄金知识是否**排在了最前面（Top-1/Top-2）**。
* *计算逻辑：* 分析前 $k$ 个 Chunk 中有用 Chunk 的比重，并结合其排名位置施加惩罚。防止“Lost in the Middle”现象。


4. **上下文召回率 (Context Recall - 测 C 覆盖 Ground Truth)：**
* *含义：* 评估系统召回的材料（C），是否完整覆盖了回答该问题所需的全部黄金知识点。
* *计算逻辑：* 检查标准答案中的每个核心知识点，有多少比例能在线上召回的 C 中被找到。如果得分低，说明**向量数据库检索漏掉了关键信息（召回率不足）**。
