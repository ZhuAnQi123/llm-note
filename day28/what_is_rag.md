# RAG（检索增强生成）
是一种结合**信息检索**与**大语言模型生成能力**的技术框架.  
核心思想是：**让模型在生成答案前，先从外部知识库中查找相关信息，再基于这些信息生成更准确、更新、更可靠的回答**。

---

### 一、为什么需要 RAG？
传统大语言模型（如 GPT）存在几个痛点：
- **知识截止日期**：训练数据截止后的事件不了解。
- **幻觉问题**：可能编造不存在的事实。
- **缺乏专业/私有知识**：无法访问企业内部文档、最新论文等。

RAG 通过实时检索外部知识，有效缓解这些问题。

---

### 二、RAG 工作原理（三步）

1. **检索**  
   用户提问后，系统**将问题转换为向量**（embedding），然后**在知识库（如文档、网页、数据库）中搜索**最相关的文本片段。常见技术：向量数据库 + 相似度搜索。

2. **增强**  
   将检索到的相关文本片段（上下文）与原始问题**拼接成一个增强提示**（augmented prompt）。

3. **生成**  
   将增强提示送给大语言模型，模型基于提供的上下文信息生成自然语言回答。

**示例**：  
用户问“2025 年诺贝尔物理学奖得主是谁？”  
- 检索：从最新新闻或维基百科中找到获奖者信息。  
- 增强：提示变为“根据以下信息回答：2025 年诺贝尔物理学奖授予 X，因为…… 问题：……”。  
- 生成：模型输出“根据检索结果，获奖者是 X”。

### 一些概念

这篇论文是 **RAG 的开山之作**，由 Meta AI（FAIR）的研究人员在 2020 年发表。以下是几个关键要点：

| 核心概念 | 解释 |
|----------|------|
| **参数化记忆** | 存储在神经网络权重中的知识（即预训练语言模型本身） |
| **非参数化记忆** | 外部知识库（如维基百科的向量索引），可随时检索和更新 |
| **检索增强生成** | 将检索到外部知识作为上下文，再让模型生成答案 |
| **两种 RAG 变体** | RAG-Sequence（整段生成用同一批检索文档）和 RAG-Token（每个 token 动态检索不同文档） |



### 三、RAG 的主要优势

| 优势 | 说明 |
|------|------|
| **知识更新** | 无需重新训练模型，只需更新外部知识库 |
| **降低幻觉** | 模型有据可依，减少编造 |
| **可解释性** | 可展示检索到的文档来源，增强可信度 |
| **成本较低** | 比微调大模型便宜，尤其适合动态知识 |
| **隐私可控** | 知识库可部署在企业内部，数据不外传 |

---

### 四、常见应用场景
- **企业知识库问答**（员工手册、产品文档）
- **智能客服**（基于最新政策或产品信息）
- **学术文献助手**（检索论文回答问题）
- **在线医疗/法律咨询**（确保引用权威来源）

---

### 五、RAG 与微调的区别

| | **RAG** | **微调** |
|--|---------|----------|
| 知识更新 | 实时/低成本 | 需重新训练 |
| 外部知识依赖 | 强 | 弱 |
| 推理成本 | 稍高（检索+生成） | 常规 |
| 适用场景 | 频繁变化、专有知识 | 改变风格/能力 |

两者也可以结合使用。

---

### 六、简单比喻
> 就像**开卷考试 vs 闭卷考试**  
> - 传统 LLM = 闭卷（只能靠记忆，容易记错或过时）  
> - RAG = 开卷（先查参考书，再组织答案，更可靠）
