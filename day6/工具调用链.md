# 🔧 OpenAI 工具调用（Tool Calling / Function Calling）学习指南

 OpenAI 工具调用（官方称为 **Tools**）

---
[版本说明](https://developers.openai.com/api/docs/guides/migrate-to-responses)：
- 旧版使用 `functions` 和 `function_call` 参数，**现已弃用**
- 新版统一使用 **`tools`** 和 **`tool_choice`** 参数
- 新项目推荐使用 **Responses API** 而非 Chat Completions API

---

## 📚 二、小白入门：什么是“工具调用”？

### 为什么需要这个？
> 核心要点：**模型只负责根据你传入的tools“建议”调用哪个函数和传什么参数，真正的执行由你的代码完成**

大模型有三个天然短板，工具调用正好解决：

| 大模型的短板 | 工具调用的解决方案 |
|-------------|-------------------|
| 不知道实时信息（如股价、天气） | 调用外部 API 获取最新数据 |
| 不擅长精确计算 | 调用计算器或代码执行 |
| 无法操作你的系统 | 调用函数发邮件、建日历、写数据库 |

## 📝 四、核心参数详解

### 1. `tools` —— 定义你有什么工具

这是你告诉模型“我有哪些能力”的地方。每个工具需要包含：

```json
{
  "type": "function",
  "function": {
    "name": "get_weather",           // 函数名（唯一标识）
    "description": "获取指定城市的天气信息",  // 描述（模型根据这个判断是否调用）
    "parameters": {                   // 参数定义（JSON Schema格式）
      "type": "object",
      "properties": {
        "city": {
          "type": "string",
          "description": "城市名称，如：北京"
        }
      },
      "required": ["city"]            // 必填参数
    },
    "strict": true                    // 是否严格遵循schema（推荐开启）
  }
}
```

### 2. `tool_choice` —— 控制模型是否调用工具

| 取值 | 含义 | 使用场景 |
|-----|------|---------|
| `"auto"` | 模型自己决定是否调用（默认） | 大多数情况 |
| `"none"` | 禁止调用任何工具 | 纯聊天场景 |
| `"required"` | 必须调用某个工具 | 强制结构化输出 |
| `{type:"function","function":{"name":"xxx"}}` | 强制调用指定工具 | 明确要用某个功能 |

### 3. 模型返回的 `tool_calls`

当模型决定调用工具时，会返回：

```json
{
  "role": "assistant",
  "tool_calls": [{
    "id": "call_abc123",           // 调用的唯一ID
    "type": "function",
    "function": {
      "name": "get_weather",        // 要调用的函数名
      "arguments": "{\"city\":\"北京\"}"  // 参数（JSON字符串）
    }
  }]
}
```

### 4. 把结果返回给模型（`role: "tool"`）

执行完真正的函数后，要把结果以 `tool` 角色的消息发回：

```json
{
  "role": "tool",
  "tool_call_id": "call_abc123",   // 对应上面的id
  "content": "北京今天晴天，28°C"   // 函数执行结果
}
```

## 💻 五、代码示例（Python）

### 最简完整示例

```python
from openai import OpenAI
import json

client = OpenAI()

# 1. 定义工具
tools = [{
    "type": "function",
    "function": {
        "name": "get_weather",
        "description": "获取指定城市的天气",
        "parameters": {
            "type": "object",
            "properties": {
                "city": {"type": "string", "description": "城市名"}
            },
            "required": ["city"],
            "additionalProperties": False
        },
        "strict": True
    }
}]

# 2. 用户提问
messages = [{"role": "user", "content": "北京今天天气怎么样？"}]

# 3. 第一次调用模型
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=messages,
    tools=tools,
    tool_choice="auto"
)

message = response.choices[0].message

# 4. 检查模型是否想要调用工具
if message.tool_calls:
    for tool_call in message.tool_calls:
        # 解析参数
        args = json.loads(tool_call.function.arguments)
        city = args["city"]
        
        # 5. 你的代码真正执行函数（这里是模拟）
        weather_result = f"{city}：晴天，28°C"
        
        # 6. 把结果加入对话
        messages.append(message)  # 先加模型的回复
        messages.append({
            "role": "tool",
            "tool_call_id": tool_call.id,
            "content": weather_result
        })
        
        # 7. 再次调用模型，生成最终回复
        final_response = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=messages
        )
        
        print(final_response.choices[0].message.content)
```

## 🔄 六、新旧 API 对比

| 特性 | 旧方式 (Chat Completions) | 新方式 (Responses API) |
|-----|--------------------------|----------------------|
| 参数名 | `functions`（已弃用） | `tools` |
| 内置工具 | 不支持 | 支持网络搜索、文件搜索、Computer Use |
| 推荐程度 | 存量项目可用 | **新项目推荐使用** |

> 官方明确：Responses API 是 Chat Completions 的超集，代表了 OpenAI 上构建智能体的未来方向

## ⚠️ 七、常见陷阱与最佳实践

### 陷阱1：以为模型会真的执行函数
❌ 错误理解：模型会自动调用 API  
✅ 正确理解：模型只返回“建议”，你来执行

### 陷阱2：参数描述写不清楚
```json
// ❌ 太模糊
"city": {"type": "string"}

// ✅ 清晰具体
"city": {"type": "string", "description": "城市的中文名称，如：北京、上海"}
```

### 陷阱3：忘记处理异常
模型可能传错参数，记得加验证和 try-catch。

### 最佳实践清单
- [ ] 为每个函数写清晰的 `description`（模型靠这个决定是否调用）
- [ ] 设置 `"strict": true` 和 `"additionalProperties": false` 让输出更可靠
- [ ] 参数用 `description` 说明格式要求
- [ ] 在生产环境中做参数验证，不要盲目信任模型输出
- [ ] 考虑并行调用多个工具的场景

## 🎯 八、学习路径建议

1. **先跑通一个简单示例**（如上面的天气查询）
2. **理解 JSON Schema**（定义参数格式）
3. **尝试多工具场景**（让模型从多个工具中选择）
4. **学习 Structured Outputs**（更严格的输出控制）
5. **升级到 Responses API**（获得内置工具支持）
