# 向量数据库

在 AI / LLM 应用中，很多内容都会被转成 **向量（embedding）**，比如：

- 一段文本
- 一张图片
- 一条用户行为
- 一篇文档

这些向量不是「完全相等」比较，而是要回答：

> **“哪个最像？”**  
> **“语义上最接近的是谁？”**

传统数据库（MySQL / Redis）**不擅长**做这种高维相似度搜索，  
而 **Pinecone 就是为这个场景而生的**。

## Pinecone

**Pinecone** 是一个专门用来做 **向量数据库（Vector Database）** 的云服务。

> 适合：**做产品、RAG、ChatBot、团队项目**
> 最核心的能力是：

### ✅ 向量存储 + 相似度搜索

### 常见使用场景

- 🧠 LLM / ChatGPT 类应用
  - RAG（Retrieval-Augmented Generation）
  - 问自己文档的聊天机器人
  - 企业知识库搜索

- 🔍 语义搜索
  - 搜“意思接近”的内容，而不是关键词完全匹配
  - 比如：
    > “怎么请年假” ≈ “年假申请流程”

- 📄 文档 / 内容推荐
  - 相似文章推荐
  - 相似商品、视频、课程

- 🧑‍💻 用户画像 & 行为相似度
  - 找“和这个用户行为最像的用户”

### 使用

1. 注册 & 创建 API Key（一次性）

1. 打开 <https://www.pinecone.io>
1. 注册账号
1. 复制 `API Key`
1. 选择一个 `Environment`

---

2. 安装 Pinecone SDK

```bash
pip install pinecone-client
```

---

3. 初始化 Pinecone 客户端

```python
import pinecone

pinecone.init(
  api_key="YOUR_API_KEY",
  environment="YOUR_ENV"
)
```

---

4. 创建 Index（只需一次）
> 注意，dimension = embedding 向量的长度，是模型决定的。你选择的模型会有特定的dimension

```python
pinecone.create_index(
  name="demo-index",
  dimension=1536,
  metric="cosine"
)
```

参数说明：

- `dimension`：embedding 维度（必须和模型一致）
- `metric`：
  - `cosine` 常用于文本
  - `euclidean`非文本 / 数值特征；或者图像，语音

---

5. 获取 Index 对象

```python
index = pinecone.Index("demo-index")
```

---

6. 插入向量（Upsert）

```python
index.upsert(vectors=[
  ("doc-1", [0.1, 0.2, 0.3], {"text": "今天天气不错"}),
  ("doc-2", [0.2, 0.1, 0.4], {"text": "我喜欢学习 AI"})
])
```

- ✅ 每个vectors向量：
  - 有 `id`
  - 有 `values`
  - 可带 `metadata`（非常重要）

---

7. 查询相似向量

```python
query_vector = [0.1, 0.2, 0.25]

result = index.query(
  vector=query_vector,
  # 注意 top_K意思是最多返回2个
  top_k=2,
  include_metadata=True
)

print(result)
```

返回的是：

```json
matches: [
  {
    "id": "doc-1",
    "score": 0.93,
    "metadata": { "text": "今天天气不错" }
  }
]
```

👉 **Pinecone 直接帮你和真实内容连好了**

---

Pinecone 强大的地方（新手一定要知道）

- ✅ **Metadata 过滤**
这里你可以填写任意你需要检索的条件，就像是你在普通数据库进行sql检索一样。比如说：

```python
filter={
    "category": {"$eq": "tech"},
    "age": {"$gte": 18},
    "status": {"$in": ["active", "premium"]},
}
```

- 👑 **Namespace** 命名空间  
用来控制只搜索当前用户的数据

注意：
1. 每个索引最多 1000 个 namespace
2. 删除 namespace 后，空间不会立即释放（需要等待后台清理）
```py
import pinecone

pc = Pinecone(api_key="xxx")
index = pc.Index("my-index")

# 1. 不同用户的数据写入不同 namespace
index.upsert(
    vectors=[("id1", [0.1, 0.2, ...], {"title": "我的文档"})],
    namespace="user_12345"  # 用户隔离
)

index.upsert(
    vectors=[("id2", [0.3, 0.4, ...], {"title": "另一个用户的文档"})],
    namespace="user_67890"
)

# 2. 查询时只搜索指定用户的空间
results = index.query(
    vector=[0.15, 0.25, ...],
    namespace="user_12345",  # 只搜这个用户
    top_k=10
)

# 3. 删除整个用户的数据（超实用！）
index.delete(delete_all=True, namespace="user_12345")```

## FAISS

**FAISS（Facebook AI Similarity Search）** 是一个 **向量相似度搜索库**，
**核心能力只有一个，但非常强：给你一堆向量，帮你快速找最像的**

✅ **高维向量的相似度搜索（最近邻搜索）**

比如：

- 给 100 万个向量
- 用一个新向量去查
- FAISS 能很快找出最相似的 Top‑K 个

---

### FAISS 经常被用在：

- 🔍 **语义搜索**
- 🤖 **RAG / LLM 检索（ChatGPT + 自有数据）**
- 📄 **文档、图片相似度查找**
- 🎯 **推荐系统**
- 🧪 **研究 / 原型验证**

---

### FAISS 的特点（优点）

- ⚡ **非常快**（C++ 实现，支持 GPU）
- 🆓 **完全开源、免费**
- 🧠 **算法丰富**（IVF、HNSW、PQ 等）
- 🧑‍💻 **本地使用**（Python / C++）

### FAISS 基本使用文档（本地 / 入门版）

安装 FAISS

CPU 版（最常用，推荐新手）

```bash
pip install faiss-cpu

2. 一次完整的最小使用流程（MVP）

### 整体流程图

    原始数据（文本）
      ↓
    Embedding 模型（如 OpenAI / 本地模型）
      ↓
    向量（numpy array）
      ↓
    FAISS Index
      ↓
    相似度搜索

---

### 第一步：准备向量数据

FAISS 不关心文本，只关心**数字向量**。

```python
import numpy as np

# 假设你已经有 embedding
vectors = np.array([
  [0.1, 0.2, 0.3],
  [0.2, 0.1, 0.4],
  [0.9, 0.8, 0.7],
]).astype("float32")
```

⚠️ 注意：

- 必须是 **float32**
- 所有向量 **维度必须一致**

---

### 第二步：创建 FAISS 索引

### 最简单的方式（暴力搜索）

```python
import faiss

#  vectors.shape[0]代表向量的个数
# vectors.shape[1]代表向量的长度，也就是dimension
dimension = vectors.shape[1]
index = faiss.IndexFlatL2(dimension)
```

含义：

- `IndexFlatL2`：
  - L2 距离（欧式距离）
  - 精确搜索
  - 简单但慢（数据大时）

---

### 第三步：把向量加进索引

```python
index.add(vectors)

print(index.ntotal)  # 向量数量
```

---

### 第四步：做相似度搜索

```python
query = np.array([[0.1, 0.2, 0.25]]).astype("float32")

k = 2
distances, indices = index.search(query, k)

print(indices)    # 相似向量的 index
print(distances)  # 距离
```

📌 返回结果含义：

- `indices`：最相似向量在原数组中的位置
- `distances`：距离（越小越像）

---

### FAISS 中你必须自己做的事情

FAISS 只返回 **index**，你通常需要：

```python
docs = [
  "今天天气不错",
  "我喜欢学习 AI",
  "向量数据库很好用"
]

result_docs = [docs[i] for i in indices[0]]
```

### FAISS VS Pinecone

FAISS **不是数据库**，更像是一个 **底层搜索引擎库**：

| 对比        | FAISS              | Pinecone        |
| ----------- | ------------------ | --------------- |
| 形态        | 库（Library）      | 云数据库        |
| 部署        | 自己管理           | 托管            |
| 持久化      | 要自己处理         | 内建            |
| 扩展 & 运维 | 自己搞             | 平台负责        |
| 适合        | 原型 / 本地 / 大厂 | 产品 / 中小团队 |

👉 **FAISS 更“底层”，更工程化** FAISS 只关心：**向量长啥样、像不像、快不快**
👉 **Pinecone 更“成品”，更省心**
